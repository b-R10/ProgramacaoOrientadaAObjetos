Abstracao
  concentrar em informaçoes relevantes

Encapsulamento
  Metodos getters e setter
  Seperar o comportamento interno de uma classe da interface disponibilizada para 
  os clientes que acessam esta classe

Modularidade
  Agrupar objetos em modulos coesos e independentes
  Alta coesao e baixo acoplamento
  Coesao: medida da diversidade dos topicos abordados por uma entidade (em POO, 
  entidade é classe)
  Classe coesa: foca em um determinado aspecto do sistema

Hierarquia
  Organizar abstracoes em hierarquias quando necessario

Modelagem por meio da UML

Classe
  Modelo abstrato usado para definir objetos com atributos e metodos
  Deve ser escrita em uma linguagem de POO, seguindo as regras e usando os comandos 
  e estruturas de linguagem 
  Deve ter um nome que seja facilmente associavel ao modelo qua a classe representa

Objeto
  Possui:
    Identidade - cada objeto é único
    Estado - valor de um atributo em um determinado instante
    Comportamento - métodos

  Criando objetos:
    Aviao umAviao = new Aviao();

  Utilizando objetos:
    umAviao.ligar();
